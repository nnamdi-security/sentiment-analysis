# Sentiment Analysis Projects

A collection of AI Development programme assignments (Turing Tech LLC, in partnership with the Enugu State Government) applying LLM-based sentiment, emotion and brand-perception analysis to real customer comments collected from social media.

## 1. Mr. Biggs (Nigeria) — LLM Comparison

A group assignment exploring how different LLMs handle the same detailed sentiment-analysis prompt on real customer comments about **Mr. Biggs**, a Nigerian restaurant franchise.

I led this assignment: I wrote the prompt-optimization guide and the prompt itself, and ran it against multiple models to compare their output.

### Repo contents

| File | Description |
|---|---|
| `Sentiment Analysis Prompt Optimization.pdf` | A short guide on common mistakes people make when prompting an LLM for sentiment analysis (undefined sentiment categories, no guidance on sarcasm/slang, no output structure, etc.) and how to structure a prompt to avoid them. |
| `sentiment-analysis-prompt.txt` | The full prompt used for this analysis, plus the 52 raw customer comments (collected from social media) that were fed to each model. |
| `mr-biggs-sentiment-report/sentiment_report_chatgpt.html` | Interactive HTML sentiment report generated by ChatGPT from the prompt above. |
| `mr-biggs-sentiment-report/sentiment_report_claude.html` | Interactive HTML sentiment report generated by Claude from the same prompt. |
| `mr-biggs-sentiment-report/sentiment_report_gemini.html` | Interactive HTML sentiment report generated by Gemini from the same prompt. |

### What the prompt asked for

For each of the 52 comments (which include Nigerian pidgin, slang, emojis, and sarcasm), the prompt asked the model to:

1. Classify overall sentiment — positive, negative, neutral, or mixed
2. Identify the dominant emotion
3. Identify key topics mentioned
4. Color-code entities (location = blue, food = green, people = orange)
5. Highlight relevant parts of speech (color-coded purple)

Then, across all 52 comments, produce an overall sentiment/emotion distribution, most-discussed topics, frequently mentioned products and locations, complaints and compliments, an overall verdict on customer sentiment, and one specific, actionable recommendation — delivered as an interactive HTML report with charts and a color legend.

### Why three reports

Each HTML report (ChatGPT, Claude, Gemini) was generated from the *identical* prompt and comment set, so they can be compared side by side for accuracy, consistency, adherence to the requested format, and quality of the final recommendation.

### Viewing the reports

Download or clone the repo, then open any of the `.html` files in the `mr-biggs-sentiment-report/` folder in a browser.

## 2. Domino's Pizza — Corporate Brand Perception Report

An individual follow-up assignment applying a more advanced, corporate-style analysis to **Domino's Pizza**, built from 135 customer comments manually collected across X (Twitter), Facebook, Instagram, YouTube and TikTok.

Unlike the Mr. Biggs project, which compared how different LLMs handle the same prompt, this project is a single, executive-ready deliverable: a full Word, PDF and spreadsheet documents report combining sentiment analysis, emotion analysis, aspect-based sentiment, topic extraction, named entity recognition, complaint/praise categorization, opportunity analysis, and prioritized business recommendations.

### Repo contents

| File | Description |
|---|---|
| `dominos-sentiment-report/Dominos_Sentiment_Brand_Perception_Report.docx` | The full corporate-style report — cover page, executive summary, 17 analytical sections, 8 charts, and prioritized (immediate / medium-term / long-term) recommendations. |
| `dominos-sentiment-report/raw-comments.xlsx` | The 135 raw, manually collected customer comments used as the source data (ID, platform, post title, post date, comment date, comment text). |

### What the report covers

- Overall sentiment distribution (positive / negative / neutral / mixed)
- Sentiment trend over time
- Emotion analysis across a 20-emotion taxonomy (joy, frustration, loyalty, disgust, hope, etc.)
- Brand perception — how customers describe the brand in their own words
- Aspect-based sentiment (taste, price, delivery, customer service, staff behavior, packaging, etc.)
- Topic analysis, ranked by frequency
- Named entity recognition (locations, staff/people, competitor brands, menu items)
- Complaint analysis (late delivery, poor customer service, wrong order, cold food, price increases, etc.)
- Praise analysis (what delights customers)
- Opportunity analysis (e.g. vegan/dairy-free menu options, discontinued-item revivals, market re-entry)
- Color-coded visualizations: sentiment pie chart, sentiment trend, emotion distribution, aspect-based stacked bars, complaint/praise bar charts, a word cloud, and a geographic distribution chart
- Prioritized recommendations (0–30 days / 1–6 months / 6–12 months) mapped to executive owners (CEO, Marketing Director, CX Manager, Operations Manager)

### Methodology note

This is a small, manually assembled sample (n=135) gathered for an academic AI-development exercise, not a statistically representative survey. The report is transparent about this limitation throughout and treats its findings as directional signals for executive discussion rather than precise population estimates.

### Viewing the report

Download or clone the repo, then open `dominos-sentiment-report/Dominos_Sentiment_Brand_Perception_Report.docx` in Microsoft Word or Google Docs.
